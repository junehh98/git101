git init

git 저장소 생성

git remote -v (add, rename, delete 등도 사용 가능)
저장소 목록보기 -v는 경로까지

git clone <url>
git 서버에서 주소지정 후 저장소 복제 -> 남의 git에서 코드를 가져올때?

git commit
의미 있는 변화에 대해 기록하는 것

git config --global user.name "nickname"
현재 시스템에서 사용할 사용자 지정하기
global, local 차이

git config --global user.email "email@gmail.com"


git add .
모든 파일 및 폴더 추가 

git add<filename>
파일이름 지정해서 추가

git commit -m "<message>"
커밋할때 커밋메시지를 작성, 수정할때마다 변경된 이유를 적으면 좋음

git pull
저장소로 코드 밀어넣기

git branch
현재 내가 위치한 branch 확인

git checkout
- branch or commit 전환
- 내용 되둘리기

git merge
branch 병합 master가 권한을 가짐

git fetch
git서버에 변경사항이 있는지만 확인하고 내 컴퓨터에 데이터를 가져오지는 않음(pull과의 차이)


git remote
등록된 리모트 저장소 이름만 보여줌

git pull request
다른 개발자들과 공유하면서 수정?? 미리알기

git stash(list, -m"message", apply, drop)
작업 임시저장


git reset

git revert

git rebase

git tag
